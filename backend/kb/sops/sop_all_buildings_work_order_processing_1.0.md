# Work Order Processing SOP

## Purpose
Standardize the process for receiving, prioritizing, assigning, and completing maintenance work orders.

## Workflow

### 1. Work Order Creation
**Sources**: Online portal, mobile app, phone, email, in-person
**Information Required**: Unit number, issue description, contact info, photos, access authorization

### 2. Priority Assignment
- **P1 Emergency**: Life safety, major system failure (< 1 hour)
- **P2 Urgent**: Significant impact, habitability (< 24 hours)
- **P3 Routine**: Normal maintenance (3-5 days)
- **P4 Planned**: Scheduled maintenance (7-14 days)

### 3. Assignment
- Auto-assign based on: skill match, availability, workload, location
- Technician notified via mobile app
- Parts ordered if needed

### 4. Scheduling
- Emergency: immediate dispatch
- Urgent: same/next day
- Routine: within target timeframe
- Resident notification: 24-48 hours advance

### 5. Completion
- Work performed and documented
- Photos of before/after
- Time and materials recorded
- Resident sign-off (if present)

### 6. Follow-up
- QA inspection (10% random sample)
- Resident satisfaction survey
- Close work order in system

## Quality Standards
- First-time fix rate: 85%
- On-time completion: 90%
- Resident satisfaction: 4.5/5.0

## Documentation
All work orders require: date/time, technician, work performed, parts used, time spent, resident feedback
